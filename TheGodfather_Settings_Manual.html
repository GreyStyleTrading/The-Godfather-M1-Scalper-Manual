Timezone & Execution Times

ServerUtcOffsetHours Number Default: 0

Tells the EA your broker's timezone. 0 = auto-detect (recommended). Only change if auto-detect fails.

Example
If your broker is GMT+2, you could set this to 2. But honestly? Just leave it at 0 and let the EA figure it out.

RegularSession Text Default: "1630-2300"

The main trading hours (regular market session). Format is "HHMM-HHMM" using broker server time.

Example
"1630-2300" = 4:30 PM to 11:00 PM broker time. For NYSE (9:30 AM - 4:00 PM EST), convert to your broker's timezone.
Enable "ShowSessionDebug = true" temporarily to see exactly when the EA thinks sessions start/end.

PreMarketSession Text Default: "1100-1630"

Pre-market hours. Only used if UsePreMarket = true. Same format as RegularSession.

TradeWindowSession Text Default: "1630-1800"

The ONLY time the EA enters trades! Your "sweet spot" window. Default is first 90 minutes (highest volatility).

Example
"1630-1800" captures NY open (9:30-11:00 AM EST) if broker is GMT+5.
Best entry window is typically first 60-90 minutes after market open. That's when the big moves happen!

Strategy Settings

LogicTimeframe Dropdown Default: PERIOD_M1

KEEP THIS AT M1 for the main "First Candle" strategy! Even on M5/M15 charts, EA calculates on M1 internally.

CRITICAL: You MUST keep the chart on M1 timeframe! The EA uses M1 candle data for all calculations. If you switch the chart to M5/M15, the EA will NOT work correctly. LogicTimeframe = M1, but the actual chart timeframe must also be M1!
This keeps EA accurate to the Scarface transcript - always M1 candles for entries.

UseFirstCandle15 True/False Default: TRUE ‚úì

THE MAIN STRATEGY! Uses "First Candle" breakout from Scarface transcript:

  1. Captures first 15-minute candle after market open
  2. Waits for 5-minute candle CLOSE outside range
  3. Enters on M1 pullback/retest
Keep this ON for authentic Scarface approach!

UseOR5 True/False Default: FALSE

Alternative strategy using 5-minute opening range instead of 15 minutes. Test both to see what works for your market.

Some traders prefer tighter 5-minute range for quicker entries. Experiment!

UsePrevDay True/False Default: FALSE

Trades breakouts of previous day's high/low. Can use alongside First Candle, but EA only takes ONE trade per day.

UsePreMarket True/False Default: FALSE

Uses pre-market session high/low. Only works if broker provides pre-market data (most index CFDs don't). Generally OFF.

FirstCandleEntryMode Dropdown Default: Conservative

How aggressive your entries:

  • Aggressive: Enters when price touches level (market order)
  • Conservative: Waits for touch + M1 close in your direction
Conservative recommended! Filters false breakouts. Aggressive = earlier entry but more fakeout risk.

RetestMaxBars Number Default: 10

After breakout, EA waits this many bars for retest. Setup expires if no retest. Set 0 to never expire.

TouchToleranceTicks Number Default: 1

Ticks of "wiggle room" for retests. Default 1 tick = within 1 tick of level counts as "touch."

BreakConfirmTicks Number Default: 2

Ticks beyond range high/low for breakout confirmation. Prevents fakeouts from barely poking through.

RetestCloseTicks Number Default: 1

In Conservative mode, price must close this many ticks back inside range after touching.

StopBufferTicks Number Default: 2

Extra buffer for stop loss. Protects against wicks barely poking your stop.

RRMultiple Number Default: 2.0

Your Risk:Reward ratio. 2.0 = targeting 2x your risk. Stop 10 points ‚Üí Target 20 points.

Example
Stop Loss: 20 points ‚Üí Take Profit: 40 points (2x) with RRMultiple = 2.0

MaxTradesPerDay Number Default: 1

Maximum trades per day. Scarface style = ONE quality trade. Higher = multiple setups, but watch for overtrading.

TradeDirection Dropdown Default: Both

Filter trade direction:

  • Both: Long and short setups
  • Longs Only: Only buys (bullish bias)
  • Shorts Only: Only sells (bearish bias)

Risk Management

PositionSizingType Dropdown Default: Fixed USD Risk

How EA calculates position size:

  • Fixed Contracts: Always X lots
  • Fixed USD Risk: Risks $X per trade - RECOMMENDED
  • Balance % Risk: Risks X% of account per trade
Fixed USD Risk is best! You know exactly how much you can lose each trade.

FixedContracts Number Default: 1.0

Only for "Fixed Contracts" mode. EA always trades this many lots.

FixedUsdRisk Number Default: 100.0

Only for "Fixed USD Risk" mode. EA calculates lot size so you lose exactly this if stop is hit.

Example
FixedUsdRisk = $100, Stop = 20 points ‚Üí EA sizes so 20 points = $100 loss

RiskPercent Number Default: 1.0

Only for "Balance % Risk" mode. Risks this % of account per trade. Compounds as account grows.

TickValue Number Default: 5.0

Fallback if broker doesn't report tick values. Usually auto-detected.

MinCalculatedVolume / MaxCalculatedVolume Number Default: 0.0 (off)

Optional caps. Min = smallest lot size. Max = largest lot size. 0 = disabled.

SLType (Stop Loss Type) Dropdown Default: Retest Candle

Where to place stop loss:

  • Retest Candle: Below/above retest candle (SCARFACE METHOD)
  • Range %: X% of opening range from entry
  • ATR Multiple: X √ó ATR from entry
  • Fixed %: X% of price from entry
  • Fixed Points: Fixed ticks away
  • Opposite Range: At opposite side of opening range
Retest Candle = Scarface method! Tight stops at retest candle low/high for max R:R.
If using trailing stops (UseTrailingStop = TRUE) or Supertrend, your actual loss may EXCEED FixedUsdRisk! EA sizes based on INITIAL stop, but trailing methods can move stop further from entry. Example: Supertrend moves stop 50 points away but you sized for 15 points = 2.5x planned risk!

SLValue Number Default: 35.0

Value for Range %, ATR Multiple, Fixed %, or Fixed Points modes. Ignored for Retest Candle.

TPType (Take Profit Type) Dropdown Default: Scarface

How to calculate take profit:

  • Scarface: Uses HOD/LOD, pivot levels, psychological numbers (SMARTEST)
  • Risk Reward: Simple R:R multiple
  • Range %: X% of opening range
  • ATR Multiple: X √ó ATR from entry
  • Fixed %: X% of price from entry
  • Fixed Points: Fixed points
Scarface mode finds natural targets (previous highs/lows, round numbers) at least your minimum R:R away!

TPValue Number Default: 2.0

Value for Range %, ATR Multiple, Fixed %, Fixed Points modes. Ignored in Scarface mode (uses RRMultiple as minimum).

SLTP_ATR_Length Number Default: 19

ATR lookback period for ATR-based stops/targets. 19 is common for intraday.

UseHighLowOfDayTarget True/False Default: TRUE ‚úì

In Scarface mode, target previous High of Day (longs) or Low of Day (shorts) if at least minimum R:R away.

UsePivotTargets True/False Default: FALSE

In Scarface mode, consider daily pivot points (PP, R1-R3, S1-S3) as targets.

UsePsychologicalTargets True/False Default: TRUE ‚úì

In Scarface mode, round to "clean" price levels (335, 340, 345). Markets love round numbers!

PsychStep / PsychStepAuto Number / True-False Default: 1.0 / TRUE

Controls psychological rounding. PsychStepAuto = TRUE lets EA auto-detect step size.

ConfirmClosePercent Number Default: 65.0

For Scarface mode. Retest candle must close in top X% of range (longs) to qualify. Filters weak retests. Recommended: 65-70 for indices.

MinWickTicks Number Default: 0

Minimum wick size (ticks) on retest candle. 0 = disabled. Helps filter doji/small candles.

UseCandleCloseStop True/False Default: TRUE ‚úì

If TRUE, EA closes position if price closes beyond conceptual stop (even if broker stop not hit).

With trailing stops or Supertrend, candle close stops may trigger at worse price than initial stop, resulting in larger losses than FixedUsdRisk. Position sizing uses INITIAL stop, but trailing methods widen effective stop.

MinStopDistanceTicks Number Default: 15

Minimum stop distance (ticks). Trade rejected if calculated stop closer. Prevents tiny stops on volatile moves.

StopManagementMode Dropdown Default: Fixed SL/TP

How EA manages stops after entry:

  • Fixed SL/TP: Stops don't move (SCARFACE DEFAULT)
  • Candle Close: Uses candle-close invalidation
  • Supertrend: Uses Supertrend to trail (requires UseSupertrend = true)
Supertrend mode can cause losses LARGER than FixedUsdRisk! EA sizes based on INITIAL Supertrend value, but Supertrend moves with price. If Supertrend moves 40 points away but you sized for 15 points, you could lose 2-3x planned risk!

Trailing Stop

UseTrailingStop True/False Default: FALSE

MASTER SWITCH! When FALSE, stops stay fixed. When TRUE, stops trail using chosen method. Scarface didn't use trailing stops - he held to target.

IMPORTANT: Trailing stops can cause losses EXCEEDING FixedUsdRisk! Position sized based on INITIAL stop at entry. As trailing stop moves, distance from entry to stop can INCREASE. Example: FixedUsdRisk = $100 with 20-tick stop. If trailing moves stop to 50 ticks away and price reverses, you lose $250 instead of $100!
Default is OFF because Scarface style holds to target. Only enable if you want to lock in profits as price moves!

TrailingStopType Dropdown Default: ATR

If UseTrailingStop = TRUE:

  • ATR: Trails by X √ó ATR behind price
  • Chandelier Exit: Uses Chandelier Exit formula

TrailingATR_Length / TrailingATR_Multiplier Number / Number Default: 16 / 3.0

For ATR trailing: ATR period and multiplier. Higher multiplier = wider trail (stops less often but gives back more profit).

TrailingMinStopDistanceTicks Number Default: 15

Minimum distance trailing stop must stay from entry. Prevents stops getting too tight.

Even with this minimum, trailing stops can still end up further from entry than initial stop! This only prevents stop being TOO CLOSE, not too far. Risk can still exceed FixedUsdRisk if price moves significantly before trailing stop adjusts.

Chandelier Exit Settings Group

Settings for Chandelier Exit trailing:

  • CE_AtrPeriod: ATR lookback (default: 34)
  • CE_AtrMultiplier1: Primary level (default: 9.0)
  • CE_AtrMultiplier2: Secondary level (default: 19.5)
  • CE_LookBackPeriod: High/low lookback (default: 22)
  • CE_UsePrimaryLevel: Use 9√ó (TRUE) or 19.5√ó (FALSE) multiplier
  • CE_DrawOnChart: Show Chandelier lines visually

Supertrend Settings

UseSupertrend True/False Default: FALSE

Enables Supertrend indicator for stop management. Only works if StopManagementMode = "Supertrend." Advanced feature - most users should leave OFF.

CRITICAL: Supertrend can cause losses MUCH LARGER than FixedUsdRisk! Here's why: EA calculates position size based on Supertrend value AT ENTRY. But Supertrend is dynamic - it moves as price moves. If price trends strongly, Supertrend can end up 40-60+ points from entry, while you sized for only 15-20 points. If price reverses to hit Supertrend line, you could lose 2-4x planned risk. Example: FixedUsdRisk = $100, initial Supertrend at 15 points away. Price trends, Supertrend moves to 45 points away. Reversal hits Supertrend = you lose $300. Use extreme caution!

ST_ATR_Length / ST_Multiplier Number / Number Default: 10 / 3.0

Supertrend ATR period and multiplier. Formula: (High+Low)/2 ¬± (Multiplier √ó ATR)

ST_BufferTicks Number Default: 0

Extra buffer beyond Supertrend line for stops.

Adding buffer makes stop even FURTHER from entry, increasing potential loss beyond FixedUsdRisk!

ST_UpdateOnNewBarOnly True/False Default: TRUE ‚úì

Only update Supertrend on new bars (prevents repainting/mid-bar changes). Recommended TRUE.

ST_DrawOnChart True/False Default: TRUE ‚úì

Visually draw Supertrend line on chart to see where stops are.
